How to setup custom activity

Introduction

Step by step guide to getting up and running with a basic Custom Activity

Step 1

Go to this page and create new configuration or pick one of the existing configurations.

  • Key - Custom activity Unique key. Must match your activity key as defined when created in App Center.
  • Type - String property representing the type of activity. Select REST for default activity and RESTDECISION for custom split activity.
  • Name - Name of activity (i.e. 'Send Email', 'SMS', 'Create Salesforce Lead'), as shown in the activity list.
  • Description - Description of what the activity does and what purpose it fulfills.
  • Icon - String representing the icon for this activity. Defaults to images/icon.png when not provided.
  • Small Icon - String representing the small display icon for this activity. Defaults to images/iconSmall.png.
  • Splits - Contains an array of objects that define the custom splits. Each object contains label and key.
  • EditUrl - Endpoint for the UI displayed to marketers while configuring this activity.
  • Steps - Contains an array of objects that define the steps that the user may navigate through when configuring the custom activity. Each object contains label and key.
  • Height and Width - Properties that defines the height and width of the iframe containing the configuration UI.

Note: Key and EndpointUrl will be used in next step.

Step 2

Create a package containing a Journey Builder Activity in Salesforce Marketing Cloud App Center. A documentation for this task can be found here: Create a Marketing Cloud App.

  • Select NO for SSO and Account Integration.
  • The Key value specified in the configuration will be used as the Unique Key!
  • The EndpointUrl value specified in the configuration will be used as the Endpoint URL!
  • Category: Message
  • Select “This application and other installed applications” for the Public Extension

Step 3

Go into Journey Builder and the Custom Activity should appear.

Note: There is some caching done by the browser that could prevent your app from showing up in Journey Builder. Going back to activities list and reloading the page usually helps to solve this issue.

What’s next?

  • Same configuration can be used for different custom activities.
  • Configuration can be changed at any time (journey builder reload required)
